2. Test navigation components:
   - **Desktop navbar**: Should collapse to hamburger menu on mobile
   - **Sidebar navigation**: Should slide out on mobile (drawer)
   - **Breadcrumbs**: Should truncate or scroll on mobile
   - **Footer**: Should stack vertically on mobile
3. Review and fix navigation template:
   ```html
   <!-- Ensure mobile menu toggle exists -->
   <nav class="navbar">
     <button class="navbar-toggler lg:hidden"
             type="button"
             data-toggle="collapse"
             data-target="#navbarMenu"
             aria-label="Toggle navigation">
       <span class="navbar-toggler-icon"></span>
     </button>
     <div class="collapse navbar-collapse" id="navbarMenu">
       <!-- Menu items -->
     </div>
   </nav>
   ```
4. Test table responsiveness:
   ```bash
   # Find all tables
   grep -r "<table" /Users/saidamenmambayao/apps/madaris-ms/src/apps/*/templates --include="*.html"
   ```
   - Add responsive wrapper:
   ```html
   <div class="table-responsive">
     <table class="table">
       <!-- Table content -->
     </table>
   </div>
   ```
   - Or use card layout on mobile:
   ```html
   <div class="hidden md:block">
     <!-- Desktop table -->
   </div>
   <div class="block md:hidden">
     <!-- Mobile card view -->
   </div>
   ```
5. Test forms on mobile:
   - Input fields should be full width on mobile
   - Labels should be above inputs (not side-by-side)
   - Select dropdowns should be native on mobile
   - Date pickers should use native mobile datepickers
   ```html
   <div class="form-group">
     <label for="field" class="block mb-2">Label</label>
     <input type="text"
            id="field"
            class="w-full px-3 py-2 border rounded">
   </div>
   ```
6. Check touch target sizes:
   ```css
   /* Ensure minimum touch target size */
   .btn, .link, .checkbox, .radio {
     min-height: 44px;
     min-width: 44px;
   }
   ```

   - **Detail pages** (Madrasah detail, Student profile):

     - Action buttons should be easily tappable
   - **Form pages** (Create/Edit forms):
     - Fields should be full width
     - Submit buttons prominent and easy to tap
   - **Dashboard pages**:
     - Stat cards should stack on mobile
     - Charts should be scrollable horizontally if needed
